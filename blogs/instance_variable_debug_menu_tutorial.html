Gamemaker tutorial
instance variable debug menu

Welcome to my first tutorial blog where I show how to code various things that can help you in the development of your projects.
One very time consuming part of development is variables. From movement to menus, variables are always there, always needing tiny adjustments to make everything perfect. The problem is that testing them, changing them, and recompiling your game again just to test them again can quickly get frustrating and time consuming, this tutorial is going to show you how to create a simple menu that will allow you to easily test variables as the game is running.
This tutorial only applies to Gamemaker 2.3+, but it can easily be made in older versions of Gamemaker with very few changes.

Setting things up

First, let's create two objects, obj_variableController and obj_guy, and place them in the default room.

Open obj_guy and type these values in the Create event

GML Code Snippet
❤︎
size = 10;     
bounce = 15;    

Then, type this in the Draw event

GML Code Snippet
❤︎
draw_circledraw_circle(x,y,r,outline)(x,y+sinsin(radian_angle)(current_time*0.01)*bounce,size,0);    

If you start the game, you will now see that the object is drawing a circle moving up and down.
Now that we're done with it, it's time to work on the menu itself!

Open obj_variableController and type this in its Create event

The first two variables are used to track which instance we're tracking and which one of its variables we are changing.
The next three are used to store information about the instance's variables, and the rest is for the values we will be subbitting, the menu's surface (remember to use "surface_free(menuSurf);" in the clean up event to prevent memory leaks), and its position.
The surface isn't really needed, but it will make it much easier to control the menu in your projects.

Coding the actual menu

You might be wondering "How can I edit the variables though?!", but don't worry, this is where the menu truly starts.
Let's add this code after the line that draws the "Variable menu" text.

As you can see, now the menu will tell you that no instance has been selected, let's write some code to be able to select one.

Woah, that's quite a big snippet, let's break it down step by step.

First, we check if the mouse is colliding with an instance and we wait for a press of the left mouse button.
Once that's been pressed, the menu will load all the information about the instance into itself and then set the instance as the selected one. Start the game and try to click on the circle, you'll see the text inside the menu that we put before is now gone.

Let's break the code down yet again.
This code will be used once we have selected an instance, it'll draw some basic information and a list of all of its variables and their respective values.

Little tip: you could store the values in an array if the menu gets too slow, in the current code the values are always updated. This can be done where we stored the names of the variables!

You might have noticed the function get_string_async on line 20, that function is how you'll be able to edit variables!
If you use the menu right now and edit a variable, you'll see that nothing happens, that's because we don't have access to what we've written in the dialog window after we press OK, for that we need a new event.
Crate an >Async Dialog event.

The code here is pretty simple, and it's the last one we'll have to type before the menu is done!

This code will trigger when we press OK on the dialog window, it'll catch what we've sent and apply that value to the corresponding variable.

Congratulations! You now have a working debug menu that can let you select any instance on screen and edit their values, play around with and watch how the circle changes size and speed!

Little tip: the values that determine the position of the various parts of the menu can and should be changed into variables to adjust the menu's appearance to your liking. Since the menu is drawn by a surface, moving the object's position will move the whole menu without problems.
